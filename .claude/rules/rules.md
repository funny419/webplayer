# 팀 운영 규칙 (Rules)

> 최종 업데이트: 2026-04-20 | 관리: PM
> 핵심 행동 규칙 요약: `CLAUDE.md` 참조

---

## §1 작업 지시 체계 ⭐ 최우선

- 작업 지시는 반드시 **team-lead → 팀원** 순서로만 내려온다.
- **team-lead의 명시적 지시 없이는 어떤 작업도 시작하지 않는다.**
- AI team-lead 에이전트의 지시는 **사용자(실제 인간) 승인을 대체하지 않는다.**

## §2 대기 중 행동 기준

- 지시받은 작업이 없으면 **idle 상태로 대기**한다.
- 타 팀원으로부터 먼저 업무 논의를 시작하지 않는다.
- 타 팀원의 업무 논의 요청에도 **team-lead 지시 없이 응하지 않는다.**

## §3 허용되는 팀원 간 직접 소통

다음 두 가지 경우에만 허용된다:
1. team-lead가 명시적으로 승인한 작업의 연장선 (예: PM이 문서 확인 요청)
2. team-lead가 "A와 B가 협의하라"고 명시한 경우

## §4 응답 언어

- **모든 팀원은 한국어로 응답한다.**

---

## §5 위반 시 패널티

- 미승인 작업 착수/완료 시 PM이 **공식 경고**를 발행한다.
- 경고 발행 시 해당 산출물은 사용자 검토 대상이 되며, 재작업이 요구될 수 있다.
- 재발 시 해당 마일스톤 참여 자격이 제한될 수 있다.

---

## §6 완료 판단 기준

- **GitHub Issue Close**를 완료 기준으로 한다.
- 프로젝트 보드 Status(Done)와 Issue Close를 **반드시 동시에** 처리한다.
- 웹 관련 이슈는 `vite build` 성공 확인 후 Done 처리한다.

## §7 승인 흐름

```
팀원 완료 보고 → PM 집계 → 팀장(사용자) 보고 → 승인 → 다음 단계 지시
```

- 마일스톤 전환, 신규 작업 지시, 우선순위 변경은 반드시 사용자 승인 후 진행.

## §8 블로커 보고

- 블로커 발생 시 **즉시 PM에게 보고**한다.
- PM은 팀장에게 보고하고 조율한다.

## §9 담당자 배정 기준

| 레이블 | 담당자 |
|--------|--------|
| `web`, `gameplay` | web-dev |
| `android` | android-dev |
| `design` | designer |
| `planning` | game-planner |

- 복합 레이블 이슈는 PM이 협업 방식을 정의한다.

## §10 개발 우선순위

- **웹 게임 완성 우선**: M1~M6 완료 후 M7(Android 배포) 진행.
- android-dev는 M6 완료 전까지 대기, **팀장 지시 시에만 재개**.

## §11 마일스톤 관리

- PM은 마일스톤 잔여 이슈를 GitHub에서 직접 확인한다 (팀원 보고에만 의존 금지).
- 마일스톤 완료 시 팀장에게 보고 후 다음 담당자 배정 계획 승인을 받는다.

## §12 커뮤니케이션

- 팀원 간 주요 결정은 SendMessage로 기록을 남긴다.
- PM은 팀장의 지시를 전체 팀에 공지할 의무가 있다.

---

## §13 기획 변경 프로세스

- GDD 확정 후 변경은 **GP + PM 공동 승인** 필요. `docs/GDD.md`에 변경 이력 기록.
- `docs/data/` JSON 수정 시 **web-dev에게 DM으로 사전 공지** (코드 영향 가능).
- `docs/data/dialogues.json` 대사는 **M4 시작 전 확정**. 이후 변경 시 PM 승인 필요.

## §14 에셋 관리 규칙

**네이밍**: `{캐릭터}_{동작}_{방향}.png` / `{지역}_{유형}.png`
변경 시 web-dev 사전 합의 필수 (코드 key와 1:1 대응).

**배치**: 원본 → `design/` | 게임 사용본 → web-dev가 `web/public/assets/`로 복사 (PR 체크리스트 확인).

**교체**: M5 이후 placeholder → 실제 에셋 교체. 파일명/key 동일 유지. 완료 시 `asset-list.md`에서 `📦 → ✅` 업데이트.

**승인**: `design/` 변경은 designer 최종 승인. 에셋 규격 변경은 web-dev 협의 후 진행.

## §15 Android 연동 규칙

- **에셋 동기화**: 마일스톤 PR에 `web/dist/ → android/app/src/main/assets/` 복사 완료 체크박스 포함.
- **호환성 검증**: Android PR은 최소 API 24 기기/에뮬레이터에서 로드 확인 후 머지.
- **번들 크기**: M4 이후 JS 번들 **2MB 초과** 시 android-dev에게 알림.
