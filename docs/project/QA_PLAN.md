## 2. 버그 관리 프로세스

### 버그 발견 시 즉시 처리 절차

1. **버그 발견** → team-lead에게 즉시 보고
2. **보고 형식**

   | 항목 | 내용 |
   |------|------|
   | 버그 등급 | `[BUG🔴]` / `[BUG🟡]` / `[BUG🟢]` |
   | 현상 | 어떤 증상인지 |
   | 재현 조건 | 어떻게 재현하는지 |
   | 관련 파일·라인 | 알 경우 기재 |
   | 예상 원인 | 추정되는 원인 |

3. **project-manager** → 즉시 Projects에 등록 (상세 내용 포함)
4. **수정 완료 후** → project-manager가 이슈 본문에 해결 내용 + 커밋 해시 기재 후 Done 처리

### QA 담당 역할

| 담당 | 역할 |
|------|------|
| code-reviewer | QA_PLAN.md 2절 체크리스트 기반 코드 정적 검증 |
| web-dev | 실제 빌드 후 동작 검증 (`vite preview` 실행) |
| game-planner | GDD 기준 기능 완성도 확인 |

## 3. 코드 집중 검증 영역

code-reviewer 의견 기준.

| 영역 | 검증 내용 |
|------|----------|
| 발사체 충돌 | projectiles↔enemies 충돌 판정 누락 없는지 확인 |
| 보스 소환 적 등록 | 소환 적이 EnemyGroup에 정상 등록되는지 확인 |
| 지역 전환 리소스 정리 | 씬 전환 시 이전 씬 오브젝트·이벤트 리스너 완전 정리 여부 |
| 대화 ↔ 전투 충돌 | dialogueActive 플래그와 전투 입력 차단 로직 연동 |
| 저장 / 로드 무결성 | 저장 후 로드 시 플레이어 상태(HP·퀘스트·인벤토리) 왕복 일치 |

**지역 전환 스트레스 테스트:** 동일 지역 입출 5회 반복 후 크래시 없음 확인
**보스 처치 흐름 확인:** 보스 4종(고블린 왕·리치·화염 드래곤·어둠의 기사) 순차 처치
